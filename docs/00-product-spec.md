# 시원칸 제품 스펙 v0.1

## 1. 제품 정의

**시원칸**은 지하철 이용자가 “지금 어느 칸에 타야 가장 쾌적한가?”를 3초 안에 결정하도록 돕는 모바일 웹앱이다.

기존 지하철 앱처럼 모든 정보를 보여주는 것이 아니라, 사용자의 상황에 맞는 **추천 결론**을 먼저 제공한다.

## 2. 핵심 사용자

### 1차 타깃
- 서울/수도권 출퇴근 MZ 직장인·대학생
- 지하철을 자주 타지만 기존 지하철 앱은 복잡하거나 구식이라고 느끼는 사용자
- 더위/추위/혼잡에 민감한 사용자

### 2차 타깃
- 교통약자/임산부/체력 약한 사용자
- 수도권 지하철 초보자
- 외국인 관광객

## 3. 핵심 문제

- 같은 열차 안에서도 위치별 체감 냉방이 다르다.
- 약냉방칸 위치를 매번 외우기 어렵다.
- 혼잡하면 객실 온도와 체감 불쾌감이 올라간다.
- 기존 앱은 기능은 많지만, “그래서 지금 어디 타야 하는가?”를 바로 알려주지 않는다.

## 4. 핵심 가치 제안

> “노선도 말고, 어디 탈지만 알려줘.”

- 더위형: 더 시원하고 덜 붐비는 칸 추천
- 추위형: 약냉방/중앙부/덜 추운 위치 추천
- 혼잡회피형: 덜 붐비는 칸 또는 다음 열차 대기 추천
- 밸런스형: 혼잡도, 냉방, 하차 동선을 균형 있게 추천

## 5. MVP 기능 범위

### 필수 기능
1. 소셜 로그인: Supabase Auth 기반 Google/Kakao/Apple 우선
2. 비로그인 익명 사용: 첫 추천은 로그인 없이 가능
3. 사용자 성향 선택
   - 더위형
   - 추위형
   - 혼잡회피형
   - 밸런스형
4. 출발역/도착역/노선/방향 입력
5. 추천 칸 표시
6. 피해야 할 칸 표시
7. 추천 이유 2~3개 표시
8. 데이터 출처/신뢰도 배지 표시
9. 사용자 피드백 저장
   - 좋았어요
   - 더웠어요
   - 추웠어요
   - 붐볐어요
10. 자주 타는 경로 저장

### 후순위 기능
- 공유 카드 생성
- 다음 열차 비교
- 날씨 기반 추천 문구 강화
- Toss Apps in Toss SDK 연동
- AI 기반 피드백 요약

## 6. 데이터 출처 배지

| 배지 | 의미 |
|---|---|
| 실측 | 공식 API의 칸별 혼잡도 사용 |
| 통계 | 과거/시간대 통계 기반 |
| 추정 | 공공데이터+규칙 기반 추정 |
| 커뮤니티 | 사용자 피드백 기반 |

## 7. 추천 결과 UX

### 예시

```text
10번째 칸 앞쪽 추천

시원함 92 · 혼잡 낮음 · 하차 동선 보통

이유
- 객실 끝 쪽이라 냉방 체감이 강한 편이에요.
- 현재 시간대 기준 중앙부보다 덜 붐빌 가능성이 높아요.
- 도착역 하차 동선 손해가 크지 않아요.

피하기: 5~6번째 칸
데이터 기준: TMAP/SK 칸별 혼잡도 · 신뢰도 높음
```

## 8. 중요 표현 가이드

교통약자석 관련 표현은 주의한다.

- 금지: “노약자석에 앉으세요.”
- 허용: “객실 끝 쪽은 냉방 체감이 강한 편이에요. 교통약자석 이용 배려는 지켜주세요.”

실시간 데이터가 없는 구간에서는 확정 표현 금지.

- 금지: “지금 3번 칸이 가장 덜 붐빕니다.”
- 허용: “시간대 패턴상 3번 칸이 쾌적할 가능성이 높아요.”
