# 역 검색 데이터 정합성 수용 테스트

## 목적

사용자가 실제 화면에서 발견한 `구남역 2호선` 노출 오류를 재발 방지한다. 역 검색은 Coolcar 서비스 범위인 수도권 전철 역만 반환해야 하며, 기존 핵심 검색어(`강남`, `여의도`, `역삼`)의 자동완성 품질은 유지되어야 한다.

## 대상

- 화면: 배포 URL `https://coolcar-sigma.vercel.app/` 홈 화면 역 검색 자동완성
- API: `GET /api/stations/search?q={검색어}&limit={n}` 및 필요 시 `line` 필터
- 데이터: `src/lib/stations.ts` 정적 역 마스터 및 DB fallback 결과

## 공통 수용 기준

1. 검색 결과는 수도권 전철 서비스 범위 내 역만 포함한다.
2. 비수도권 도시철도 역(부산/대구/대전/광주 등)은 검색 결과에 나오면 안 된다.
3. 역 이름, 노선, 운영기관, 좌표가 서로 모순되지 않아야 한다.
4. 사용자 입력의 `역` 접미사 유무와 공백 차이는 동일하게 처리한다. 예: `구로`, `구로역`, `구로 디지털`.
5. API 응답은 `200 OK`, JSON schema `{ stations: Station[], source: string }`를 유지한다.
6. 화면 자동완성 결과와 API 결과의 역명/노선이 불일치하면 실패로 본다.
7. 기존 정상 검색어의 결과 순위와 포함 여부가 깨지면 회귀로 본다.

## 출시 차단 조건

- `구` 검색 결과에 `구남역 2호선` 또는 다른 비수도권 역이 노출된다.
- `구로디지털단지역 2호선`, `구로역 1호선`, `구반포역 9호선` 중 하나라도 검색되지 않는다.
- `강남`, `여의도`, `역삼` 검색에서 기존 핵심 역이 누락된다.
- API는 정상이나 실제 브라우저 자동완성에 잘못된 역이 표시된다.
- 검색 결과 선택 후 추천 플로우가 출발/도착 역명 또는 노선을 잘못 전달한다.

## API smoke 예시

```bash
BASE_URL=https://coolcar-sigma.vercel.app
curl -s "$BASE_URL/api/stations/search?q=%EA%B5%AC&limit=50" | jq '.stations[] | {name,line,lat,lng}'
curl -s "$BASE_URL/api/stations/search?q=%EA%B5%AC%EB%A1%9C%EB%94%94%EC%A7%80%ED%84%B8&limit=10" | jq '.stations'
curl -s "$BASE_URL/api/stations/search?q=%EA%B5%AC%EB%A1%9C&limit=10" | jq '.stations'
curl -s "$BASE_URL/api/stations/search?q=%EA%B5%AC%EB%B0%98%ED%8F%AC&limit=10" | jq '.stations'
curl -s "$BASE_URL/api/stations/search?q=%EA%B0%95%EB%82%A8&limit=10" | jq '.stations'
curl -s "$BASE_URL/api/stations/search?q=%EC%97%AC%EC%9D%98%EB%8F%84&limit=10" | jq '.stations'

```

## 브라우저 확인 절차

1. 새 시크릿 창에서 `https://coolcar-sigma.vercel.app/` 접속.
2. DevTools Network를 열고 `stations/search` 요청을 보이도록 필터링.
3. 출발역 입력창에 `구` 입력.
4. 화면 후보와 API 응답을 비교한다.
   - 금지: `구남역`, `구서역`, `구암역(대전)`, `대구/부산/광주/대전 도시철도 역`.
   - 필수: `구로디지털단지역 2호선`, `구로역 1호선`, `구반포역 9호선`.
5. `구로디지털단지역`을 선택한 뒤 입력값/노선 라벨이 유지되는지 확인.
6. 도착역 입력창에서 `구반포` 검색 후 `구반포역 9호선` 선택.
7. 추천 실행 시 네트워크 요청 payload에 선택한 역명/노선이 그대로 전달되는지 확인.
8. 동일 방식으로 `강남`, `여의도`, `역삼` 검색이 기존처럼 동작하는지 확인.

## QA 판정

- 위 STS-001~STS-018이 모두 통과해야 역 검색 데이터 정합성 수정본을 수용한다.
- 하나라도 출시 차단 기준에 해당하면 배포 보류 및 역 마스터/DB fallback/filter 로직 재검토가 필요하다.
